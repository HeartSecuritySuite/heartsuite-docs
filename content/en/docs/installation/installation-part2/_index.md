You must augment the initial HeartSuite allowlist database so that essential startup and shutdown programs will be permitted to run as needed. Use the dashboard review tools as the primary method for Phase 1 and allowlisting. A Python script named **hs-os-boot-setup.py** is included as one initial option (completing Phase 1 with dashboard verification). It must be run as root using Python3:

```bash
# sudo python3 /.hs/sys/hs-os-boot-setup.py
```

--OR--

```bash
# su # if you’re not logged in as root
# python3 /.hs/sys/hs-os-boot-setup.py
```

Each time you run this script (as one initial method), it will scan the HeartSuite log, described below, and the kernel log for permission error messages generated by HeartSuite. The script will then use these permission messages to run a HeartSuite tool named **hs-allowlist-manager** to build appropriate allowlist entries. The dashboard provides guided review and suggested next steps as the primary approach after each cycle.

After building these records, you must reboot and then run the script again because new permission error messages will likely be generated. In fact, the script will direct you to reboot and run it again with the following message:

After three cycles of rebooting and running the script, the allowlist database should be complete for startup and shutdown programs. If so, the script will print the message, “Great! Your OS and its related programs are now whitelisted.”

Depending on your distro, it may take four or five cycles of rebooting and running the script in order to complete this part of the Installation.

## If the Script Doesn't Progress or Completes Without Success

If after several runs the script doesn't show the success message or you encounter errors:
- Check the HeartSuite log: `cat /.hs/sys/hs-activity-log.txt` for permission errors.
- Add missing programs using the dashboard review tools (or `hs-allowlist-manager` for advanced cases; see whitelisting section).
- Ensure you're running as root and the kernel is loaded (the dashboard also shows status).
- Review via dashboard tools for HeartSuite-related events.

> [!TIP]
> This script analyzes logs to auto-allowlist startup programs—run it multiple times until it reports success.

Note that the **hs-os-boot-setup.py** script also writes certain messages to standard output. As you run the this script, you will see error messages directed to standard error by the underlying calls to **hs-allowlist-manager**. The **hs-os-boot-setup.py** script also adds its own messages but to standard out. These messages indicate when permission is substituted to access a directory rather than a file within it. By default, the **hs-os-boot-setup.py** script adds directory permission instead of file permission whenever it detects that a particular file may be temporary in nature. The script assesses this quality based on whether the name of the file contains digits or capital letters. Additionally, the **hs-os-boot-setup.py** script evaluates directory names that contain digits as temporary, as well.

> [!WARNING]
> Repeating these cycles (or equivalent reviews via the dashboard) is vital before changing to Secure mode. If you skip building the initial allowlist enough times in Setup Mode, your system may hang on boot/shutdown after switching to Secure Mode!
